# Diccionario de datos

## Descripción

El diccionario de datos es el documento donde se define de manera estructurada toda la información que maneja el sistema. Su objetivo principal es describir cada una de las entidades del sistema, junto con sus atributos, tipos de datos, restricciones y significado dentro del contexto del proyecto.

Este documento permite estandarizar la información antes de ser transformada en un modelo de base de datos, garantizando coherencia entre el análisis, el diseño y la implementación del sistema.

---

## Contenido del diccionario de datos

El diccionario de datos incluye:

- Entidades principales del sistema
- Atributos de cada entidad
- Tipo de dato de cada atributo
- Descripción funcional de cada campo
- Reglas o restricciones de los datos

---

Este archivo contiene toda la estructura detallada de los datos del sistema y sirve como base para la creación del modelo entidad-relación, el modelo relacional y la normalización de la base de datos.

---

## Importancia

El diccionario de datos es fundamental porque:

- Permite definir correctamente la estructura de la base de datos
- Facilita la creación del modelo entidad-relación
- Sirve como base para la normalización
- Mejora la organización y consistencia de la información del sistema
