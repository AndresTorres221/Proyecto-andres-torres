## ¿Qué hice?

Se avanzó significativamente en la fase de diseño del sistema del proyecto “Banco de Alimentos”, trabajando de forma estructurada en GitHub y VS Code.

Se construyeron y documentaron las principales tablas de diseño y trazabilidad del sistema, cubriendo arquitectura, datos, componentes, interfaz y comportamiento.

Se desarrollaron las siguientes estructuras:

- Tabla de entidades externas y su relación con casos de uso y módulos.
- Tabla de arquetipos del sistema relacionados con casos de uso y requerimientos funcionales.
- Tabla de componentes del sistema relacionando arquetipos, RF e interfaces.
- Tabla de nodos del sistema con protocolos, requerimientos no funcionales y componentes.
- Tabla de entidades del modelo con reglas de negocio asociadas.
- Tabla de relaciones del modelo entidad-relación con cardinalidades y reglas.
- Tabla de modelo relacional con claves primarias, foráneas y forma normal.
- Tabla de clases con métodos relacionados a casos de uso y RF.
- Tabla de secuencias del sistema con flujos principales y alternos.
- Tabla de estados del sistema con eventos, acciones y requerimientos funcionales.
- Tabla de pantallas del sistema con roles, casos de uso y wireframes.
- Tabla de prototipos con relación entre wireframes, interacciones y pantallas destino.

Además, se estructuró la carpeta de diseño en GitHub, organizando adecuadamente cada fase del sistema y manteniendo trazabilidad entre análisis, diseño y futuras etapas de implementación.

---

## ¿Qué problema encontré?

Durante el desarrollo del proyecto se presentaron algunos inconvenientes:

- Conflictos en Git al intentar sincronizar cambios entre el repositorio local y remoto.
- Confusión inicial en el orden de archivos dentro de GitHub debido a la numeración de los documentos.
- Dificultades en la organización de imágenes y su correcta ubicación dentro del repositorio.
- Necesidad de estandarizar nombres de entidades y consistencia en la documentación.

---

## ¿Cómo lo resolví?

- Se resolvieron conflictos de Git mediante `git pull`, edición manual de archivos y posterior `git add`, `commit` y `push`.
- Se entendió que GitHub ordena los archivos alfabéticamente, no por orden lógico del proyecto.
- Se organizó la carpeta de imágenes dentro de `docs/diseño/imagenes/` para mantener orden visual.
- Se estandarizó la estructura de las tablas y se mejoró la coherencia entre RF, CU y módulos del sistema.
